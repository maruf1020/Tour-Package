    1. Introduction

    TOUR PACKAGE is a online tour package booking system with some good features, an user could know about several tour packages by chatbot. The owner wouldn't be needed to reply the users, rather the system will automatically reply when someone ask something in chatbot. Moreover, one could book a tour package, see the details including google map location of the tour packages, could search his favorite tour package, and would be able to pay online for the package. User would need to sign up and sign in to do these activities, third party log in system would be available also.
    This project would be very helpful for those who want to book tour package by sitting at home. He would be able to get the answers with in a second if he has any query. The owner of the TOUR.

    3.1 User Registration

  • Use Case 

     

    Actors 

    New Tourist

     

    Purpose 

    Register the user information 

    Overview 

    Given that a user has downloaded the mobile application, or accessed the website then the user should be able to register into the system. The user must provide Name, Password, email.

    Type 

    Primary 

     

    Typical course of events: 

    1. Open the application or website
    2. Enter all the information
    3. Click on register

  • System Response:

    1. Adds the new user to the database
    2. Takes him to the login page
    3. Treats the new user as a returning customer afterwards

  • ALTERNATIVE COURSE OF EVENTS: 

    1. Email not match with regex, show error: “Please provide a valid email” 
    2. Password is less than 6 digits, show error: “Password must include at least 6 digits”   

  • 3. Any field left blank, show error: “All the fields must be filled up”

    Use Case 

    User Login 

    Actors 

    New tourist

    Purpose 

    Login the user 

    Overview 

    Given that the user is already registered, the user will be able to login by providing the email and password. 

    Type 

    Primary function

    3. Any field left blank, show error: “All the fields must be filled up”

    3.7 Use Case: Rate and Review Product

      1.  

        Use Case 

        Rate and Review Tour 

         

        Actors 

        User

        Purpose 

        Rate and Review previously taken tour packages. 

        Overview 

        From the Product page the user can click on “Review product” button. This will pop up a window with 5 empty stars and a comment box. The user will have to click on any star which will signify the rating based on one’s index of the star. Th user can additionally add a review comment into the box and submit the rating. 

        Type 

        Primary function 

         

         

        COURSE OF EVENTS:  

        Actor Actions: 

        1. Select a star 
        2. Optionally type a comment 
        3. Click on submit 

        System Response: 

        1. Takes the review 
        2. Recalculate overall review and update the rating 
        3. Adds the comment into the past reviews list ALTERNATIVE COURSE OF EVENTS: 

        I) None
